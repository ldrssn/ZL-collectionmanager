# Implement Loading Screen

The goal is to show a rich loading screen while the app is fetching data from the database, instead of showing an "empty collection" screen prematurely.

## Proposed Changes

### [Component] [LoadingScreen](file:///C:/_ldrssn/Github/ZL-collectionmanager/components/LoadingScreen.tsx) [NEW]
- Create a new functional component `LoadingScreen` that renders the HTML and CSS provided by the user.
- It will include the SVG animation and the pulsing "Kollektion wird geladen" text.

### [App.tsx](file:///C:/_ldrssn/Github/ZL-collectionmanager/App.tsx) [MODIFY]
- Add a new state variable `itemsLoading` initialized to `true` (or `false` depending on auth state).
- Update the `loadItems` useEffect to set `itemsLoading` to `true` before fetching and `false` after.
- Modify the rendering logic:
    - If `authLoading` OR `itemsLoading` is true, show the `LoadingScreen`.
    - Only show the "empty collection" screen if `itemsLoading` is false and `items.length === 0`.

## Verification Plan

### Automated Tests
- Run `npm run dev` and observe the initial load.
- Ensure the `LoadingScreen` appears before the collection (or empty collection screen) is displayed.

### Manual Verification
- Verify the animation and styles match the user's requirements.
- Confirm that the "empty collection" screen only appears after loading finishes if no items are found.
