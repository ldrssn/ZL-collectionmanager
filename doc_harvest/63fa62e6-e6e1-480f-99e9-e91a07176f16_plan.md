# Database Migration Plan: LocalStorage to Firebase (Germany)

## Goal
Migrate the application from `LocalStorage` to **Firebase** (Google Cloud) and host the frontend on **Vercel**.
This satisfies your requirements:
- **Free Tools**: Firebase Spark Plan (Free) + Vercel Hobby (Free).
- **Germany Hosting**: You will select `europe-west3` (Frankfurt) for your data.

## Capacity Analysis (Firebase Free Tier)
| Resource | Firebase Free Limit | Your Usage Estimate | Status |
| :--- | :--- | :--- | :--- |
| **Database (Firestore)** | 1 GB | ~50 MB (Text) | ✅ **Safe** |
| **File Storage** | 5 GB | ~1.2 GB - 3 GB (Photos) | ✅ **Safe** (Better than Supabase) |
| **Reads/Day** | 50,000 | ~50,000 (Risk if usage is heavy) | ⚠️ **Caching Needed** |

> [!TIP]
> **Why Firebase?** The 5GB storage limit is perfect for your photo needs. To stay within the "Reads" limit, we will ensure the app caches data so it doesn't re-download your 500 items every time you open the menu.

## User Review Required
> [!IMPORTANT]
> **Action Items**:
> 1. Go to [console.firebase.google.com](https://console.firebase.google.com).
> 2. Create a Project. Disable Google Analytics (simpler setup).
> 3. **CRITICAL**: When setting up **Firestore** and **Storage**, select location **`europe-west3` (Frankfurt)** to keep data in Germany.
> 4. Enable **Authentication** (Email/Password).
> 5. Create a Web App in settings and copy the `firebaseConfig` object.
